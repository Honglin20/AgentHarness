# 基础量化

5 分钟完成一次模型量化分析。你会看到框架如何自动分析项目、配置参数、运行量化，最终生成一份包含精度对比和层级别 QSNR 的可视化报告。

## 分析项目结构 @analyzer

如果你手动用量化工具（比如 bitx），第一步得搞清楚你的项目结构：模型类在哪、权重文件是什么格式、数据集叫什么。

通常你会这样做：

1. 打开项目目录，翻找 `models/` 或 `nets/` 文件夹
2. 搜索继承 `nn.Module` 的类，确定模型类名和模块路径
3. 找到 `.pt` 或 `.pth` 权重文件
4. 确认数据集名称和加载方式

工具层面，你会用到：
- **grep** — 搜索 `nn.Module` 子类定义，定位模型类名
- **glob** — 查找 `.pt` / `.pth` 权重文件和已有适配器
- **bash** — 执行快速验证脚本

手动做这些很繁琐，而且容易漏掉细节（比如有多个模型文件时不知道该选哪个）。

最终你需要整理出这些信息：

| 字段 | 说明 |
|------|------|
| model_class | 模型类名，如 `ResNet18` |
| model_module | 模块导入路径，如 `models.resnet` |
| dataset | 数据集名称，如 `CIFAR-10` |
| weights_path | 权重文件路径 |
| weights_exist | 权重文件是否存在 |

## 配置量化参数 @configurator

知道项目结构后，你要写一个量化适配器（adapter），告诉 bitx 怎么包装你的模型。

手动流程：

1. 确定量化位数（INT8? INT4? 混合精度?）
2. 写适配器代码，指定哪些层需要量化、哪些保持 FP32
3. 如果某些信息不确定（比如你用了自定义数据集），还得翻文档查怎么配置
4. 组装完整的量化运行命令
5. 跑一下适配器，验证代码能正常 import

工具层面：
- **ask_user** — 如果检测到不确定的参数，向你确认量化位数等关键设置
- **write_file** — 写入适配器代码
- **bash** — 验证适配器是否可导入

最终输出一份配置：

| 字段 | 说明 |
|------|------|
| adapter_path | 适配器文件路径 |
| cli_command | 完整的量化运行命令 |
| w_bits / a_bits | 权重/激活比特数 |
| block_size | 分块量化粒度 |

## 执行量化脚本 @runner

配置写好后，就该跑量化了。这一步的核心是调用 [Quantizer](api/quantizer.md) 进行实际的量化操作。

手动的话，你在终端执行上一步生成的命令，然后盯着输出看精度有没有掉：

1. 运行量化脚本，等待完成
2. 对比 FP32 基线精度和量化后精度
3. 检查每层的 QSNR（量化信噪比），找到掉精度最严重的层
4. 如果精度不达标，回到上一步调参数重试

工具层面：
- **bash** — 执行量化脚本，捕获输出

关键指标：

| 字段 | 说明 |
|------|------|
| status | 运行状态：success / error |
| fp32_accuracy | FP32 基线精度 |
| quant_accuracy | 量化后精度 |
| accuracy_delta | 精度差值 |
| worst_layer | QSNR 最差的层 |
| worst_qsnr_db | 最差层的 QSNR 值 |

## 保存诊断数据 @diagnostic_saver

量化跑完后，把中间数据存下来，方便后续分析。

手动的话你会：
1. 把每层 QSNR、block 级误差等整理成 JSON
2. 按时间戳或实验名建目录存好
3. 确保数据格式对后续报告生成工具友好

工具层面：
- **bash** — 写入诊断 JSON 文件

输出：

| 字段 | 说明 |
|------|------|
| diagnostic_dir | 诊断数据目录路径 |
| status | 保存状态 |

## 生成报告 @report_painter

最后一步，把所有数据变成人能看懂的报告。

手动的话你会用 matplotlib 或 tensorboard 画图表，或者用 Excel 整理表格。但更好的方式是直接生成包含交互式图表的报告：

工具层面：
- **render_chart** — 绘制可视化图表（精度对比柱状图、QSNR 分布图等）
- **read_text_file** — 读取诊断数据
- **bash** — 辅助数据处理

输出的报告包含精度对比、层级别 QSNR 分布、以及文字分析。

## 总结

上面就是一次完整的量化分析流程。如果你手动做，每一步都要写代码、查文档、盯终端——至少半小时。

但这个 workflow 已经把所有步骤串好了：

1. **analyzer** 自动发现项目结构，省去手动翻目录
2. **configurator** 生成量化配置，不确定的参数直接问你
3. **runner** 执行量化并记录精度，不需要盯终端
4. **diagnostic_saver** 持久化中间数据，格式自动对齐
5. **report_painter** 生成可视化报告，直接看结论

点击左下角**「试一试」**，输入你的项目路径，框架会自动走完整个流程。
